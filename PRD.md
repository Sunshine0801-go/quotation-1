## 2. 제품 개요

`SteelBid Comparator`는 강구조 견적 비교 업무를 위한 웹 기반 견적 비교 시스템이다.  
프로젝트별 BOQ, 업체별 견적, 환율, 발주범위, 분리/사급 발주 시나리오를 통합 관리하고, 견적 비교표·Summary·Query·추천 대시보드까지 연결하는 것을 목표로 한다.

본 ver3에서는 기존 견적 비교 기능에 더해 다음 기능을 강화한다.

1. 발주방식 시나리오를 실무 기준 6개 유형으로 재정의한다.
2. Summary 화면에 `추천 발주방식`과 추천 사유를 명확히 표시한다.
3. BOQ 관리, 견적 업로드·매칭, 견적 비교표, 비교 Summary, Query 관리 화면에 저장 버튼을 배치한다.
4. 견적 업로드·매칭 화면에 `업로드 초기화` 버튼을 추가하고, 초기화 전 확인창을 제공한다.
5. 외주일괄의 범위를 명확히 정의한다.

## 3. 배경 및 문제 정의

### 3.1 기존 업무 문제

강구조 외주 견적 비교는 단순 총액 비교만으로는 적정 발주 판단이 어렵다.

주요 이슈는 다음과 같다.

- 업체별 견적 Scope가 다르다.
- 형강/강재 구매, 도료 구매, 제작, 납품, 설치 포함 여부가 다르다.
- 통화와 Incoterms가 다르다.
- BOQ 항목별 누락, 단위 불일치, 이상 단가 확인이 필요하다.
- 사급 전환 시 발주금액은 낮아질 수 있으나 수행 책임과 리스크가 달라진다.
- 보고/결재 시 왜 특정 발주방식이 추천되는지 설명이 필요하다.

### 3.2 ver3 개선 방향

ver3에서는 단순 업체 순위가 아니라, `발주방식별 시나리오 비교`와 `추천 사유 설명`을 강화한다.

특히 Summary 화면에서 아래 내용을 한눈에 확인할 수 있어야 한다.

- 추천 발주방식
- 추천 조합
- 예상 총액
- Budget 대비 절감률
- 추천 이유
- 확인 필요 리스크

## 6. 발주방식 정의

ver3에서 사용하는 발주방식은 총 6개이다.

| No. | 발주방식 | 정의 | 비고 |
|---:|---|---|---|
| 1 | 외주일괄 | 철골 제작/납품 + 형강구매 + 도료구매 + 철골제작물 설치를 모두 포함 | 수행 책임이 외주사에 가장 많이 집중됨 |
| 2 | 철골 제작/납품 | 철골 제작 및 납품 중심 발주 | 설치 포함 여부는 실제 프로젝트 기준 확인 필요 |
| 3 | 외주일괄+형강사급 | 외주일괄 범위 중 형강/강재 구매를 발주자가 별도 사급 | 형강 단가 경쟁력 반영 가능 |
| 4 | 철골 제작/납품+형강사급 | 제작/납품 발주에 형강/강재를 발주자 사급으로 전환 | 제작사 Scope와 사급 Scope 구분 필요 |
| 5 | 외주일괄+형강사급+도장사급 | 외주일괄 범위 중 형강/강재와 도장/도료를 사급으로 전환 | 가격 절감 가능성은 크나 관리 Interface 증가 |
| 6 | 철골 제작/납품+형강사급+도장사급 | 제작/납품 기준에 형강 및 도장 사급을 반영 | 실제 현장 설치/도장 작업 Scope 구분 필요 |

### 6.1 외주일괄 기준 정의

본 시스템에서 `외주일괄`은 다음 Scope를 모두 포함하는 발주방식으로 정의한다.

- 철골 제작
- 철골 납품
- 형강/강재 구매
- 도료 구매
- 철골제작물 설치

따라서 외주일괄은 금액 비교뿐 아니라 수행 책임, 공정 Interface, 납기 리스크, 현장 설치 리스크까지 함께 검토해야 한다.

### 7.5 ⑤ 견적 비교표

#### 목적
BOQ 세부 항목 기준으로 업체별 단가, 금액, Budget 대비율을 비교한다.

#### 주요 기능

- 발주방식 선택
- 업체별 단가 비교
- 업체별 금액 비교
- Budget 대비율 표시
- 이상 단가 알람 표시
- Excel 출력
- PDF 출력
- 저장
- Summary 화면 이동

#### 발주방식 탭

견적 비교표에는 아래 6개 발주방식 탭을 제공한다.

1. 외주일괄
2. 철골 제작/납품
3. 외주일괄+형강사급
4. 철골 제작/납품+형강사급
5. 외주일괄+형강사급+도장사급
6. 철골 제작/납품+형강사급+도장사급

#### 표시 컬럼

고정 컬럼:

- 공종
- 세부내역
- 단위
- 수량
- Budget(USD)

업체별 반복 컬럼:

- 단가(USD)
- 금액(USD)
- vs Bgt%

#### Budget 대비율 계산

```text
Budget 대비율(%) = 업체 금액 / Budget 금액 × 100
```

표시 기준:

| 상태 | 표시 |
|---|---|
| Budget 이하 | ▼ 절감률 표시 |
| Budget 근접 | 주의 색상 표시 |
| Budget 초과 | ▲ 초과율 표시 |

#### 주요 기능

- 공종별 총액 비교
- Budget 대비율 표시
- 시나리오별 Grand Total 비교
- 추천 발주방식 표시
- 추천 사유 표시
- 주의사항 표시
- Excel 출력
- PDF 출력
- 저장
- Query 관리 화면 이동

#### Summary 발주방식 탭

1. 외주일괄
2. 철골 제작/납품
3. 외주일괄+형강사급
4. 철골 제작/납품+형강사급
5. 외주일괄+형강사급+도장사급
6. 철골 제작/납품+형강사급+도장사급

#### 추천 발주방식 카드

Summary 화면 상단에 `추천 발주방식` 카드를 표시한다.

표시 항목:

| 항목 | 설명 |
|---|---|
| 추천 발주방식 | 시스템이 추천하는 발주방식명 |
| 추천 조합 | 업체 조합. 예: Baosteel + POSCO + KCC Steel |
| 예상 총금액 | 추천 시나리오 기준 총액 |
| Budget 대비 | Budget 대비 절감/초과율 |
| 추천 이유 | 왜 해당 발주방식이 추천되는지 설명 |
| 주의사항 | Query 확인 필요, Scope 불명확 등 |

#### 추천 사유 예시

추천 발주방식: `외주일괄+형강사급+도장사급`

추천 사유:

- 일괄 공종 기준 최저 업체를 기본으로 적용한다.
- 형강/강재는 POSCO 사급 적용 시 Budget 대비 절감 효과가 있다.
- 도장/도료는 KCC Steel 사급 적용 시 Budget 대비 절감 효과가 있다.
- 외주일괄 기본 범위는 철골 제작/납품, 형강구매, 도료구매, 철골제작물 설치를 모두 포함하므로 수행 책임 관리가 명확하다.
- 단, 이상 단가 Query 회신 확인 후 최종 확정이 필요하다.

#### 주의 문구

다음과 같은 경우 Summary에 주의 문구를 표시한다.

- 이상 단가가 존재하는 경우
- 누락 항목이 있는 경우
- 단위 불일치가 있는 경우
- Budget 초과 업체가 있는 경우
- 외주일괄과 철골 제작/납품의 Scope 차이가 산식에 충분히 반영되지 않은 경우

---

### 7.7 ⑦ Query 관리

#### 목적
견적 비교 중 확인이 필요한 항목을 Query로 관리한다.

#### Query 유형

| 유형 | 설명 |
|---|---|
| 누락 항목 | 견적서에 BOQ 항목이 누락된 경우 |
| 단위 불일치 | BOQ 단위와 업체 견적 단위가 다른 경우 |
| Spec 불일치 | 요구 사양과 업체 견적 사양이 다른 경우 |
| 이상 단가 | 타 업체 대비 과도하게 낮거나 높은 단가 |
| Budget 초과 | Budget 대비 초과된 항목 |

#### 주요 기능

- Query 목록 확인
- Query 상태 필터
- Query 초안 확인
- Query 수정
- Query 복사
- 발송 완료 상태 변경
- 회신 등록
- 비교표 반영
- 저장

#### Query 상태

- 초안 대기
- 발송 완료
- 회신 완료
- 비교표 반영 완료

## 9. 계산 로직

### 9.1 기준통화 환산

```text
기준통화 단가 = 원통화 단가 × 환율
기준통화 금액 = 기준통화 단가 × 수량
```

예시:

```text
KRW 단가 38,000 × KRW→USD 0.000720 = USD 27.36
USD 27.36 × 450m = USD 12,312
```

### 9.2 Budget 대비율

```text
Budget 대비율 = 기준통화 금액 / Budget 금액 × 100
```

### 9.3 충실도

```text
충실도 = 매칭 완료 항목 수 / 전체 BOQ 항목 수 × 100
```

단위 불일치와 누락 항목은 충실도 계산 시 별도 상태로 표시한다.

### 9.4 이상 단가 알람

이상 단가는 동일 BOQ 항목에서 업체 간 단가 차이가 과도할 때 표시한다.

확정 기준 (v3.1):

```text
최저가가 2번째 최저가 대비 30% 이상 저렴한 경우 이상 단가로 판정하고 ⚠ 알람을 표시한다.
```

이상 단가로 판정된 항목은 영문 Query 초안을 자동 생성하며, 초안에는 해당 항목명, 최저가, 2위 업체명·단가, 저렴한 비율(%)이 포함된다.

확인 필요:

- 공종별 기준율 차등 적용 여부는 추후 검토 필요

### 9.5 추천 발주방식 산정

추천 발주방식은 아래 요소를 종합하여 산정한다.

| 요소 | 설명 |
|---|---|
| 총액 | Budget 대비 총금액 절감 효과 |
| 충실도 | 업체 견적의 누락/불일치 여부 |
| 이상 단가 | 비정상 단가 존재 여부 |
| Scope 적합성 | 발주방식별 포함/제외 범위 적합성 |
| 수행 책임 | 외주일괄 대비 사급 전환 시 책임 분산 정도 |
| 리스크 | 납기, PQ, Incoterms, Interface 리스크 |

추천 로직 예시:

```text
추천 점수 = 가격 점수 + Budget 점수 + 충실도 점수 - 이상 단가 패널티 - Scope 리스크 패널티
```

단, 현재 HTML 목업은 실제 점수 산식까지 완전 구현된 상태는 아니므로, 운영 시스템에서는 점수 가중치 정의가 필요하다.

- 충실도
- 납기 리스크
- Incoterms 리스크
- PQ 승인 여부

### 15.4 Query 회신 반영 이력 관리

Query 회신 전/후 단가 변경 이력을 남기면 감사 대응과 내부 검토에 유리하다.

추천 발주방식: 외주일괄+형강사급+도장사급

## 19. AI 견적 분석 어시스턴트 (v3.1 추가)

### 19.1 목적

⑥ 견적 비교 Summary와 ⑧ 추천 대시보드 화면에 `🤖 AI 견적 분석` 버튼을 배치한다. 버튼 클릭 시 현재 프로젝트의 견적 비교 데이터(BOQ 세부 비교표, 공종별 Summary, 업체별 매칭 현황, 이상 단가 후보)를 서버(API)를 통해 Claude API로 전달하여 AI 분석 결과를 화면에 표시한다.

### 19.2 처리 흐름

1. 사용자가 ⑥ 또는 ⑧ 화면에서 `AI 견적 분석` 버튼을 클릭한다.
2. 브라우저가 현재 선택된 발주방식, BOQ 비교 데이터, 업체 매칭 현황, 이상 단가 후보를 정리하여 서버 API(`/api/analyze-quotation`)로 POST 요청을 보낸다.
3. 서버(`/api` 폴더의 서버리스 함수)가 환경변수 `ANTHROPIC_API_KEY`를 사용해 Claude API(Messages API)를 호출한다.
4. Claude가 지정된 JSON 스키마에 맞춰 분석 결과를 반환한다.
5. 브라우저가 응답을 파싱하여 화면에 렌더링한다.
   - ⑥: 누락·오류 리스트, 이상 단가 항목과 영문 Query 초안
   - ⑧: 추천 발주방식 2안(발주방식·협력사 조합·예상 총액·Budget 대비율·추천 사유·유의사항)

### 19.3 출력 항목

| 항목 | 표시 화면 | 설명 |
|---|---|---|
| 누락·오류 리스트 | ⑥ | 업체별 누락/단위 불일치/이상 단가/Budget 초과 항목과 확인 상태(확인됨/확인 필요) |
| 이상 단가 항목 + 영문 Query 초안 | ⑥, ⑧ | 30% 기준 초과 항목에 대해 영문 확인 메일 초안을 자동 생성 |
| 추천 발주방식 (2안) | ⑧ | 서로 다른 발주방식·협력사 조합 2가지를 총액/Budget 대비율/추천 사유/유의사항과 함께 제시 |
| 톤앤매너 안내 문구 | ⑥, ⑧ | "본 분석은 AI 참고 의견이며, 최종 발주방식 및 협력사 선정은 담당 Buyer의 권한입니다." 문구 필수 포함 |

### 19.4 톤앤매너 규칙

- 데이터로 단정할 수 없는 값·판단은 `확인 필요` 배지로 표시한다.
- 모든 AI 분석 결과 하단에는 "본 분석은 AI 참고 의견이며, 최종 발주방식 및 협력사 선정은 담당 Buyer의 권한입니다." 문구를 표시한다.

### 19.5 이상 단가 판정 기준 (9.4절과 동일, 확정)

```text
최저가가 2번째 최저가 대비 30% 이상 저렴한 경우 이상 단가로 판정하고 ⚠ 표시 + 영문 Query 초안을 생성한다.
```

### 19.6 보안 요구사항

- `ANTHROPIC_API_KEY`는 화면(클라이언트) 코드에 절대 포함하지 않는다.
- 해당 키는 Vercel 프로젝트의 환경변수로만 저장하며, `/api` 폴더의 서버리스 함수에서만 참조한다.
- 배포는 Vercel을 전제로 한다. 정적 호스팅인 GitHub Pages에서는 `/api` 서버리스 함수가 동작하지 않으므로 AI 견적 분석 기능이 작동하지 않는다. Vercel 프로젝트 Settings → Environment Variables에 `ANTHROPIC_API_KEY`를 등록해야 기능이 정상 동작한다.
