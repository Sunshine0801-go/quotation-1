**문서 버전:** v3.0
**최초 작성일:** 2026.06.27 | **최종 수정일:** 2026.07.05
**작성 부서:** 글로벌 강구조 구매본부

> 2. 견적 분석 절차 5단계 체계화 (BOQ 대조 → 통화 확인 → 단위 통일 → 오류 식별 → 추천)

## 1. 프로젝트 개요

### 1-1. 프로젝트명
**SteelBid Comparator**
강구조 외주사·강판 Supplier 견적 비교 및 최적업체 추천 시스템

### 1-2. 목적
글로벌 강구조 구매본부 담당자가 Carbon / Stainless Steel 강판 및 강구조 외주사로부터 수신한 **다양한 양식의 견적서(엑셀·PDF)** 를 BOQ 기준으로 체계적으로 비교하고, 누락·오류를 자동 식별하여 Query를 발행하며, 데이터 기반으로 최적 업체를 추천받을 수 있도록 한다.

### 1-3. 배경 및 문제 인식
- 업체마다 견적서 양식이 제각각이어서 수작업 비교에 많은 시간이 소요됨
- BOQ 항목과 업체 견적 항목의 누락·불일치를 수동으로 확인해야 하는 비효율 발생
- 통화가 다양하여 (USD, SGD, CNY, EUR, GBP, JPY, KRW 등) 환산 오류 리스크 존재
- Budget 대비 견적 금액 비교가 별도 작업으로 진행되어 누락 리스크 존재
- 공종별 합계(Summary) 파악이 어려워 전체 견적 수준 판단이 어려움
- Painting, Material Procurement 등 일부 공종을 분리 발주 시 비교 체계가 없음
- 이상 단가 감지 기준이 없어 리스크 관리에 취약
- 업체별 Query 발행 및 회신 관리가 체계화되어 있지 않음

## 2. AI 견적 분석 어시스턴트

### 2-1. 역할 정의
본 시스템에 내장된 AI 어시스턴트는 **글로벌 강구조 구매 견적 전문 어시스턴트**로서 아래 역할을 수행한다.

> 강구조 외주사 및 강판(Carbon·Stainless Steel) Supplier로부터 수신한 견적서를 체계적으로 정리·비교하고, 누락·오류를 식별하며, 최적 업체 선정을 위한 분석 의견을 제공한다.

---

### 2-2. 견적서 분석 절차 (5단계)

#### Step 1. BOQ 대조 및 누락 확인
업로드된 견적서와 요청 내역(BOQ)을 대조하여 **누락 항목을 가장 먼저 확인**한다.

| 확인 항목 | 내용 |
|-----------|------|
| 항목 누락 | BOQ에 있으나 견적서에 없는 항목 |
| Spec 불일치 | 재질·규격·등급이 BOQ와 다르게 기재된 항목 |
| 수량 불일치 | BOQ 수량과 다른 수량으로 견적된 항목 |
| 단위 혼재 | MT, PCS, m, m², nos 등 단위가 BOQ와 다르게 기재된 항목 |

#### Step 2. 수신 통화 확인 및 기준 통화 설정
견적 업로드 시 아래 절차를 반드시 따른다.

**절차**
1. 담당자에게 "이 프로젝트의 **기준 통화**와 **적용 환율**을 알려주세요"라고 먼저 확인한다.
2. 담당자가 기준 통화와 환율을 지정하면, 모든 업체 견적을 해당 기준 통화로 환산하여 비교표를 작성한다.
3. 환율이 제공되지 않은 경우 참고용 환율을 제시하되, **"실제 적용 환율은 담당자 확인 필요"** 임을 반드시 명시한다.

**지원 통화**

| 통화 | 코드 | 주요 대상 국가 |
|------|------|----------------|
| 미국 달러 | USD | 미국, 국제 기준 통화 |
| 싱가포르 달러 | SGD | 싱가포르 |
| 중국 위안 | CNY (RMB) | 중국 |
| 유로 | EUR | 유럽 |
| 영국 파운드 | GBP | 영국 |
| 일본 엔 | JPY | 일본 |
| 한국 원 | KRW | 한국 |

**비교표 기재 기준**
- 원본 통화 단가
- 기준 통화 환산 단가
- 적용 환율 및 환율 기준일

#### Step 3. 단위 통일
단위(MT, PCS, mm, m, m², nos 등)가 혼재된 경우 **환산 기준을 명시**하고 통일한다.

| 케이스 | 처리 방법 |
|--------|-----------|
| BOQ: MT / 견적: PCS | PCS → MT 환산 기준 명시 후 통일 |
| BOQ: m² / 견적: m | 환산 불가 시 ⚠ 표시 후 Query 발행 |
| 단위 자체가 누락된 경우 | ⚠ 표시 후 업체에 확인 요청 |

#### Step 4. 오류·이상 식별 및 Query 초안 생성
계산 오류, Spec 불일치, 이상 단가 발견 시 **⚠ 표시** 후 **Query 초안을 별도 제공**한다.

**이상 단가 기준**

| 기준 | 처리 |
|------|------|
| 2번째 최저가 대비 30% 이상 저렴 | ⚠ 알람 + Query 초안 자동 생성 (필수) |
| 10~30% 저렴 | ⚠ 경고 표시 + Query 발행 권장 |
| Budget 대비 초과 | ⚠ 표시 + 사유 확인 Query 권장 |

**Query 초안 생성 규칙**
- 영문 초안을 기본으로 제공한다.
- 업체명, 항목명, 이슈 내용, 확인 요청 사항을 구체적으로 명시한다.
- 오류 유형별로 개별 Query를 생성한다 (누락 / 단위 불일치 / Spec 불일치 / 이상 단가 / Budget 초과).

#### Step 5. 분석 완료 및 최적 업체 추천
가격·조건·특이사항을 종합하여 **최적 업체 추천 의견**을 제시한다.

**종합 평가 항목**

| 평가 항목 | 내용 |
|-----------|------|
| 가격 순위 | 기준 통화 환산 총금액 기준 |
| Budget 대비 | 내역별·전체 Budget 대비 금액 비율 |
| 견적 충실도 | BOQ 항목 중 견적 제출 비율 (%) |
| Incoterms | FOB / CIF / DAP 등 물류 조건 |
| 물류비 포함 여부 | 단가에 운임 포함 여부 |
| 납기 | 업체 제시 납기 (주 단위) |
| 이상 단가 여부 | 30% 초과 알람 항목 수 |
| 분리 발주 조합 | 공종별 분리 발주 시 통합 금액 |

### 2-3. AI 출력 형식 표준

#### 출력물 ① 견적 비교표
```
No. | Item | Spec | 단위 | 수량 | Budget(USD)
  | 업체A 원본통화 단가 | 업체A 환산단가 | 업체A 금액 | vs Budget(%)
  | 업체B 원본통화 단가 | 업체B 환산단가 | 업체B 금액 | vs Budget(%)
  | ... | 비고(⚠ 표시)
```

#### 출력물 ② 환율 정보 블록
```
[환율 정보]
기준 통화  : USD
환율 기준일 : YYYY.MM.DD
적용 환율  : KRW 0.000720 / CNY 0.1378 / JPY 0.00650 / SGD 0.7421
※ 실제 적용 환율은 담당자(재무팀) 확인 필요
```

#### 출력물 ③ 누락·오류 리스트
```
[누락·오류 리스트]
No. | 항목명 | 오류 유형 | 대상 업체 | Query 초안
1   | A387 Gr.11 t=40mm | 이상 단가 (39.5%↓) | Tata Steel | Dear Tata Steel, ...
2   | A516 Gr.70 t=60mm | 견적 미제출 | POSCO | Dear POSCO, ...
```

#### 출력물 ④ 추천 의견
```
[최적 업체 추천]
추천 업체 : (업체명)
추천 근거 :
  - 가격: 전체 N개사 중 X위 ($XXX,XXX — Budget 대비 X%)
  - 견적 충실도: XX% (BOQ N개 항목 중 N개 제출)
  - Incoterms: XXX (물류비 포함/미포함)
  - 납기: XX주
  - 이상 단가: 없음 / X건 (Query 확인 필요)
유의사항 :
  - (비가격 요소, PQ 승인, 납기 리스크 등)
※ 본 추천은 분석 의견이며, 최종 선정은 담당 Buyer와 협의 후 결정합니다.
```

## 4. BOQ 구조 (실제 데이터 기준)

업로드된 **철골 자재/제작 BOQ Table**을 기준으로, 아래 3단계 계층 구조로 관리한다.

```
구분 (Level 1)
  └── 내역 (Level 2)
        └── 세부내역 (Level 3) + Spec + 수량 + 단위 + Budget
```

### 4-1. 실제 BOQ 항목 구조

| 구분 (L1) | 내역 (L2) | 세부내역 (L3) | 단위 |
|-----------|-----------|---------------|------|
| Ancillary steel works | Ancillary steel works | Galvanized Steel Ladder fabrication | m |
| | | Galvanized Steel caged Ladder fabrication | m |
| | | Galvanized handrail fabrication | m |
| | | Galvanized grating fabrication | m² |
| | | Galvanized embedded material fabrication | ton |
| | | Galvanized step grating fabrication | m² |
| Structure steel works | Shop drawing | (일식) | ton |
| | Material Procurement | Steel for building/structure | ton |
| | | Steel for pipe rack | ton |
| | Fabrication | Steel for building/structure | ton |
| | | Steel for pipe rack | ton |
| | Transportation | Steel for building/structure | ton |
| | | Steel for pipe rack | ton |
| | Steel Painting | Blast clearing | m² |
| | | Steel coating for indoor structure | m² |
| | | Steel coating for outdoor structure | m² |
| | Miscellaneous Material | High strength bolt | nos |
| | | Common bolt | nos |
| | | Anchor bolt | nos |

### 4-2. BOQ 관리 항목

| 항목 | 내용 |
|------|------|
| 입력 방식 | 직접 입력 또는 엑셀 업로드 (.xlsx, .xls, .csv) |
| 계층 구조 | 구분(L1) / 내역(L2) / 세부내역(L3) 3단계 |
| 관리 항목 | No. / 구분 / 내역 / 세부내역 / Spec / 수량 / 단위 / Budget / 비고 |
| Budget | 내역별 예산 금액 — 기준 통화 기준, 견적 비교표에 함께 표시 |
| 단위 | ton, m, m², nos 등 — 혼재 시 환산 기준 명시 |

### 5-2. 발주 범위 (Scope Split) 설정
프로젝트별로 아래 공종을 **전체 일괄** 또는 **분리 발주**로 구성할 수 있다.

| 공종 | 분리 발주 가능 | 비고 |
|------|---------------|------|
| Ancillary steel works | ○ 가능 | |
| Structure steel works — Shop drawing | ○ 가능 | |
| **Structure steel works — Material Procurement** | **✅ 분리 발주 빈번** | 별도 강판 Supplier 지정 가능 |
| Structure steel works — Fabrication | ○ 가능 | |
| Structure steel works — Transportation | ○ 가능 | |
| **Structure steel works — Steel Painting** | **✅ 분리 발주 빈번** | 별도 Painting 업체 지정 가능 |
| Structure steel works — Miscellaneous Material | ○ 가능 | |

> **분리 발주 시 동작 방식**
> - 담당자가 프로젝트 생성 시 공종별로 "일괄" 또는 "분리(별도 업체)" 중 선택
> - 분리 선택된 공종은 해당 공종 전용 견적 비교 탭이 별도 생성됨
> - Summary 뷰에서 일괄 업체 + 분리 업체 금액을 통합 합산
> - 시나리오 비교: 1위사 일괄 + 분리 업체 / 2위사 일괄 + 분리 업체 등 조합별 비교 가능

### 5-3. 견적 관리

| 항목 | 내용 |
|------|------|
| 업체 정보 | 업체명 / 국가 / 견적 통화 / Incoterms |
| 견적 입력 방식 | 엑셀(.xlsx) 또는 PDF 업로드 후 BOQ 항목에 수동 매칭 |
| 확인 항목 | 단가 / 총금액 / Budget 대비 금액 / Incoterms / 물류비 포함 여부 / 납기 / 견적 충실도 |

### 5-4. 비교·분석 항목

| 항목 | 내용 |
|------|------|
| 단가 비교 | 원본 통화 단가 + 기준 통화 환산 단가 |
| 금액 | 수량 × 단가 (기준 통화 환산) |
| Budget 비교 | 내역별 Budget 금액과 업체 견적 금액 병기 / Budget 대비 % 자동 표시 |
| 견적 충실도 | BOQ 항목 중 견적 제출 비율 (%) |
| Incoterms | FOB / CIF / DAP 등 조건 비교 |
| 물류비 | 포함 / 미포함 여부 표시 |
| 이상 단가 알람 | 2번째 최저가 대비 30% 이상 저렴 시 ⚠ 자동 표시 |
| 공종별 Summary | 7개 공종별 합계 + 단가(금액÷전체수량) + Budget 대비 % |
| 분리 발주 시나리오 | 일괄 순위별 + 분리 발주 조합별 Grand Total 비교 |

## 6. 주요 기능

### 기능 ① 프로젝트 생성 및 발주 범위 설정
- 프로젝트 생성 시 기준 통화·통화별 고정 환율·환율 기준일 등록
- 7개 공종별 일괄/분리 발주 선택
- Material Procurement 및 Steel Painting 분리 발주 시 전용 견적 탭 자동 생성

### 기능 ② BOQ 등록 및 Budget 입력
- 3단계 계층(구분 / 내역 / 세부내역) 구조로 BOQ 등록
- 항목별 Budget 금액 입력 (기준 통화 기준)
- 단위 이상 항목 즉시 표시
- 엑셀 양식 다운로드 → 작성 후 업로드 방식 지원

### 기능 ③ 견적서 업로드 및 BOQ 매칭
- 엑셀·PDF 업로드 지원 (업체별 양식 무관)
- 업체별 견적 항목을 BOQ 항목에 1:1 수동 매칭
- 매칭 후 누락 항목, 단위 불일치, Spec 불일치 자동 표시

### 기능 ④ 환율 처리
- 프로젝트 생성 시 기준 통화 및 통화별 환율 등록
- 견적 업로드 시 AI가 기준 통화·환율 확인 요청
- 환율 미제공 시 참고용 환율 제시 + "담당자 확인 필요" 명시
- 비교표에 원본 통화 단가 + 환산 단가 + 환율 기준일 병기

### 기능 ⑤ 견적 비교표 자동 생성 (세부 내역 뷰)
- BOQ 3단계 계층 기준 업체별 비교표 자동 생성
- 컬럼 구성: 단위 / 수량 / 원본 통화 단가 / 환산 단가 / 금액(수량×단가) / Budget 대비 %
- 기준 통화 환산 자동 계산 (프로젝트 고정 환율 적용)
- Incoterms·물류비 포함 여부·납기 병기
- 견적 충실도 (%) 자동 산출
- 이상 단가 ⚠ 알람 자동 표시 (2위 대비 30% 초과 시)
- 발주 방식 탭 전환: 일괄 / 자재 분리 / 도장 분리 / 자재+도장 분리

### 기능 ⑥ 견적 비교 Summary 뷰 (공종별 합계)
7개 공종별로 업체 견적 합계를 비교하는 Summary 테이블을 별도 제공한다.

| Summary 항목 | 표시 내용 |
|-------------|-----------|
| 공종명 | Ancillary / Shop Drawing / Material / Fabrication / Transportation / Painting / Misc |
| 단위 | ton (전체 철골 수량 기준) |
| 수량 | 전체 철골 수량 |
| Budget | 공종별 예산 총액 |
| Budget 단가 | Budget ÷ 전체 수량 |
| 업체별 단가 | 공종 금액 ÷ 전체 수량 |
| 업체별 금액 | 공종 세부항목 합계 |
| vs Budget | 공종별 Budget 대비 % (초과 ▲ / 절감 ▼) |
| Grand Total | 전체 합계 (분리 발주 포함 통합) |

**시나리오 비교 순서 (컬럼 배치)**
1. 일괄 발주: 1위사 → 2위사 → 3위사 (총금액 기준 오름차순)
2. 분리 발주: 1위사 + 분리 업체(자재/도장) 조합
3. 2위사 + 분리 업체 조합

### 기능 ⑦ 분리 발주 시나리오 통합 비교
- 일괄 업체 + 분리 업체 금액 합산 → 프로젝트 총 발주 금액 자동 계산
- 분리 발주 조합별 Grand Total 비교 카드 제공
- Budget 대비 절감액·절감률 자동 표시

### 기능 ⑧ 누락·오류 식별 및 Query 생성
- 누락 / 단위 불일치 / Spec 불일치 / 이상 단가 / Budget 초과 자동 감지
- 항목별 Query 초안 자동 생성 (영문 기본)
- 화면 내 Query 리스트 확인 및 초안 수정
- 발송 상태 관리 (초안 대기 / 발송 완료 / 회신 완료)
- 엑셀·PDF 출력 지원

### 기능 ⑨ 최적 업체 추천
- 가격 순위 / Budget 대비 / 충실도 / Incoterms / 이상 단가 종합 분석
- 분리 발주 시 공종별 최적 업체 조합 추천
- 추천 업체 + 추천 근거 + 유의사항 자동 생성
- "최종 선정은 담당 Buyer 협의 후 결정" 문구 자동 포함

### 기능 ⑩ 결과 출력
- 견적 비교표(세부 내역) 엑셀 다운로드
- 견적 비교 Summary 엑셀 다운로드
- 결재용 보고서 PDF (세부 내역 + Summary + 추천 의견 통합)
- 요약 리포트 자동 생성

### 기능 ⑪ 데이터 보관 및 조회
- 프로젝트·견적 DB 영구 저장
- 과거 프로젝트 검색 및 재열람
- 엑셀·PDF 개별 다운로드 병행

## 10. 주의사항 및 원칙

- 최종 업체 선정은 담당 Buyer와 협의 후 결정한다. 시스템 및 AI의 추천은 분석 의견이며 결정권은 담당자에게 있다.
- 환율 변동에 따른 금액 차이가 발생할 수 있으므로, 환율 기준일을 항상 명시한다.
- 환율이 제공되지 않은 경우, AI는 참고용 환율을 제시하되 "실제 적용 환율은 담당자 확인 필요"임을 반드시 명시한다.
- Budget은 내부 참고용이며, 외부 업체에 노출되지 않도록 출력물 설정에서 Budget 컬럼 숨김 옵션을 제공한다.
- 분리 발주 시 각 공종 담당 업체의 합산 금액을 반드시 확인 후 발주 결정한다.
- 수치는 반드시 근거와 함께 제시하며, 불확실한 값은 추정 표시 후 확인을 요청한다.
- 이상 단가 알람은 참고 지표이며, 실제 Query 발행 여부는 담당자가 판단한다.
- AI 출력물은 전문적이고 신뢰감 있는 어투를 사용하며, 과장 표현·근거 없는 평가를 지양한다.
